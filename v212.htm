V2.12

Ui-Features

❑clear attribute: Shows a "clear"-button next to the input field. A click on the button removes the value of the field.
❑barcodeFilter Forces a barcode scan and passes the value in a JavaScript-Function. In the JavaScript-Function filters can be activated.
❑The view of the toolbar in insight-mobile has been changed. The toolbar is now always displayed as a single line. To switch to the multi line behavior use: legacyToolbar
❑The rootMode: filter is now available in insight-mobile. No initial listing, a filter must be activated.
❑The tableOpen action allows client-side sorting and filtering within the table and an additional javascript function. The data of the table can be modified on load.
❑sound on validation: Plays a sound. There are two default sound files: positive and negative
❑showUserInHeader: Shows the logged in user on the top of each page.
❑heartbeatTimeout: Heartbeat request timeout in milliseconds.
❑searchPattern: The searchPattern can be configured globally.
❑textOverflow:ellipsis: List of attributes can be truncated at the end of a line.
❑js to process objects of menu actions
❑DnD action now supports flow configuration and any kind of action

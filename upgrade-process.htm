Upgrade middleware
1.stop Wildfly-Process and remove Wildfly-Service  wildfly\bin\service\service.bat uninstall

 

2.stop Kibana-Process  \kibana\bin\service.bat uninstall and remove Kibana-Service  kibana\bin\service.bat uninstall

 

3.stop Elastic-Process  and remove Elastic-Service  elasticsearch\bin\service.bat remove

 

4.rename  "insight"-folder to "insight-save"

 

5.extract middleware.zip in new and empty folder insight

 

6. -- obsolete

           elastic data migration

 copy folder nodes

         from        insight-save\elasticsearch\data\<somename>\nodes

         to                insight\elasticsearch\data\nodes

 

7.open console and start bin\elasticsearch.bat  just to make sure that process is able to run

 - first start will take some time because of datamigration.

 - stop process

 

8.install elastic-service with "\bin\elasticsearch-service.bat install"

 - make sure that service is set to automatic

 - start Elastic

 - Check Elastic in browser: localhost:9200/_cat/indices => all indices should be listet

 

9.rename babelcache.json-example to .babelcache.json  ( Dot-File! ) in folder kibana\optimize

 

10.open console and start bin\kibana.bat just to make sure that process is able to run

 - First start will take some time

 - Stop process

 

11.install kibana-service with "\bin\service.bat install"

 - make sure that service is set to automatic

 - start kibana

 - check kibana in browser: localhost/insight/kibana/ or localhost:5601

 

12. db-Files

       copy folder and containing files  from insight-save/db  to insight/db

13. Wildfly

 - adapt path for reldb and insightdb in \insight\wildfly\standalone\configuration\standalone.xml (see saved standalone.xml)

 - copy insight-example.properties to insight.properties

 - adapt entries in insight.properties  from        \insight-save\wildfly\standalone\configuration\insight.properties

 - install Wildfly-Service  bin\service\service.bat install

 - make sure that service is set to automatic

 - start Wildfly (indexing won't work till update to v2.8)

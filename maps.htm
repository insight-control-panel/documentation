Maps

Installation

To be able to create maps you need to make sure the "insight-maps.war" is deployed into the running Wildfly.

 

Inside the insight.properties file the property "map.store.folder" controls where uploaded maps are stored:

map.store.folder=D:/insight/map-store/

 

Please make sure to backup that folder regularly!

 

 

Configuration

The Insight-Map components has three menu actions to be called from any trees context menu.

Map-Administration: action: "mapCreate"

Opens the administrative view where maps can be created and edited. This menu entry should be available to administrative users only!

Map-Attachment: action: "mapAttachNode"

Opens a view where an object can be "attached" to one or many maps. Attached objects must not have a marker on the map but represent the whole map. This menu entry should be available to administrative users only!

Map-Open: action: "mapOpen"

Opens the map view for the object it got execute on. If the actionValue "allowEditMode" is set to true the user is able to create/move/delete markers on the map. Without that option users can only see existing markers.

Example menu entries:

{

 "label": "Karten-Administration",

 "icon": "icon-new-tab"

 "action": "mapCreate",

},

{

 "label": "Karte öffnen",

 "icon": "icon-arrow-right",

 "action": "mapOpen",

 "actionValues": [{

         "allowEditMode": true

 }]

},

{

 "label": "Karte Verknüpfen",

 "icon": "icon-map-marker-down",

 "action": "mapAttachNode",

}

Map Integration

To enable map-links in your tree set "clientProperties.checkForMaps: true".

V2.13

v2.13.1

❑Decimal attributes: plusMinusHidden: Using "clientProperties.decimalPointHidden" the decimal point button is hidden on the input dialog. 
❑Decimal attributes: decimalPointHidden:  Using "clientProperties.plusMinusHidden" the plus and minus buttons are hidden on the input dialog.
❑Deep Link Mobile: The insight mobile app supports the call of functions within the app
❑insight-mobile app server configuration via link. 
❑Self Service: With the self-service function, insight-mobile can be used without access data
❑Client Configuration can be scanned via barcode (config.json)

 

v2.13.2

❑remove: Deletes an object.
❑timeTrack: Time Tracker functionality.
❑selectFromTree: Default behavior changed. Attributes with selectFromTree can't be edited manually.
❑The table implementation has been rewritten for insight-web and offers some new functionality.
❑searchDelay: You can now configure the searchDelay on a global level in the config.js.
❑Maximo-Doclinks can be uploaded via Maximo-Rest-Api: document.doclink.mode=rest
❑loginParams: Additional parameters that will be automatically appended to the loginUrl, are displayed in the info dialog and are accessible in the $context object.
❑findInTree: The validation function can be used in offline and online scenario. The 'normal' variant with tree name and filter do not longer return the search-result as second parameter. The variant with tree name, filter and search query returns the search-result and only works offline.
❑existsValidator: Can be used online and offline.
❑searches -> showFields: Full text searches can be executed on specific fields. 
❑searchTrees: Defines the trees to search in.
❑detailsLink: Shows the details of an object on another tree or node
❑Menu action browser: deepLink: Opens installed apps via URL-Scheme.
❑bucketExpanded: Shows the bucket initially expanded. (insight-mobile only)
❑footerButtons: The footerButtons can be configured globally.
❑bucketMode = hidden: Buckets can be hidden

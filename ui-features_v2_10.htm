UI-Features V2.10
❑Fingerprint -> iOS Fingerprint and Face-Id login
❑Barcode options -> Sets global options for the app barcode scanner. (Android, iOS)
❑Menu, actions
❑filter -> Sets a filter on the root node
❑then -> Further action to be executed if the previous action was successful. Any number of actions can be linked, this can be archived by another "then" property.
❑createSilent -> Creates a new node item of referenced node type without a user input.
❑createForm -> Creates a new record without user input and does not save it. This record can then be used in further actions.
❑form -> Shows a dialog on which the user can make inputs.
❑confirm -> Shows a confirm dialog on which the user can decide to proceed.
❑request payloadKey -> If the request action is part of a then-chain, the result can be passed to the next action
❑geo -> Gets the current position of the device
❑Search Features
❑searches -> Search on specific attributes. (Reduces the size of the offline search model)
❑selectFromTree search -> search name to use when selecting
❑backend -> Searches can be executed on the eam database tables
❑Explorer, configure labels for tabs
❑Explorer, details view with different source tree
❑Explorer, details view with table on children with list-attributes
❑Explorer, tableOpen action
❑parentMapping: Can be used to define foreign keys to parent nodes. This is necessary to create offline objects with subobjects.
❑visibleFunction: JavaScript-Function to further limit the visibility.
❑afterSave: After saving, actions can be performed. The same actions are available as with Menu.
❑saveButtonCreateText: Label of save button on record creation views

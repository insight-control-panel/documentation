    Planning¶

    Overview

    Insight Planning allows you to show a gannt diagram based view on data from a tree-configuration.

    Configuration

    To use Planning from a context menu use the menu-action "planning":

    {
    "label": "Teamplanung",
    "action": "planning",
    "icon": "icon-new-tab",
    "actionValues": [{
        "gantt": {
            "tree": "team-workorder-planning"
        },
        "tab": {
            "label": "Planung Teams"
        },
        "table": {
            "tree": "jobtask"
        }
    }]
}

    With actionValues you can further configure the planing view:

    Gantt

    Define the used Insight tree-configuration for the data visualized in the gantt diagram.

    "gantt": {
    "tree": "team-workorder-planning"
}

    The used tree depends on where you call menu-action from:

    • from the menu of a tree All records of the root node will be shown in the gantt view. Records may be filtered out, if they are outside of the view-range and the root node of the table-tree has a temporal configuration. If you do not specify a tree, the currently active tree will be used.

    • from the contextmenu of a node The gannt-view will be opened with the right-clicked record. If you specify a tree, the id of the node will be interpreted as a record of the root node of the specified tree.

    • Expand Rows To initially expand all gantt-rows set expandRows to true.

    • absoluteZero A date that represents the absolute start for the gantt diagram. It is automatically passed to the headerFormats js function.

    Example:

    "gantt": {
    "expandRows": true,
    "tree": "team-workorder-planning"
},

    To specify different views use:

    View-Configurations

    Gantt can be shown with different view ranges and steps and precision. If not specified, gantt will use default views.

    The following view setting can be configured:

    • precision: The precision you can move the records as a ISO8601 duration. The precision cannot be defined below 1 day. Examples: "PT15M" (15 minutes), "PT1H" (1 hour), "P1D" (1 day)

    • label: The label of the view, defaults to the precision in readable format. Examples: "Year", "Week", "Month"

    • icon: The icon of the view, defaults to a calender for precisions up to a day and else a clock. Examples: "icon-calendar-31", "icon-clock3"

    • columnWidth: The width of a column in pixels. The default value depends on the precision. Examples: 50,20, 30

    • default: Set to "true" if you want to use this view as initial default.

    • range: Definition of the view-range.

      • from: The start of the view-range relative to the reference date (the reference date is initially the current day) as ISO8601 duration. The default value depends on the precision. Negative durations may be used to define a date in the past. Examples: "-P1DT1H" (1 day and 1 hour ago), "-P1M1D" (1 month and 1 day ago),
      • to: The finish of the view-range relative to the reference date. The default value depends on the precision. Examples: "P1D" (this day), "P1M" (in one month).
      • step: The amount of time to move the reference date forwards or backwards with the move-buttons. The default value depends on the precision. Examples: "P1D" (1 day), "P1M" (1 month)

    Example

    "gantt": {
    "tree": "resource-team",
    "views": [
    {
        "label": "Month",
        "precision": "P1D",
        "icon": "icon-calendar-31",
        "default": true,
        "range": {
        "from": "P0D",
        "to": "P4DT23H",
        "step": "P1D"
        }
    }
    ]
},

    Headers

    You can configure and adapt the headers shown above the gantt diagram in Insight Planning. As the default behaviour, Insight Planning will automatically decide what headers to show above your gantt diagram, based on the current view settings. You can change the actual headers shown for a given view with the following configuration:

        "views": [
      {
        "label": "LABEL_YEAR",
        "precision": "P1W",
        "headers": {
          "month": {
            "format": "return column.date.format('MMMM YYYY');"
          },
          "week": {
            "format": "return 'some String';"
          },
          "custom1": {
            "tree": "header-data",
            "scale": "week",
            "format": "var a = column.date.startOf('week'); var b = moment(absoluteZero, 'DD.MM.YYYY').startOf('week'); var diff = a.diff(b, 'week'); if(diff = 0) { diff += 1; } var hit = data.filter(function(each) { return each.properties.id == diff }); if(hit.length  0) { return hit[0].properties.activityAmount; } else { return '-' }"
          }
        }
      }
    ]

    The following properties can be configured inside each header:

    • name: The name used as a label for the header, optional

    • scale: The scale that header should use, optional. If no scale is used, the header key will be used as a scale. See list of supported scales for valid scale configurations

    • format: The format function to apply for each header cell

    • tree: The tree configuration that should be loaded as input for the custom format function, optional

    Insight Planning supports the following scales of headers:

    • millisecond

    • second

    • minute

    • hour

    • day

    • week

    • month

    • quarter

    • year

    Variables accessible in the "headerFormats" js function:

    • column: The gantt diagram column that this header cell is in.

    • currentView: The currentView that is active.

    • absoluteZero: The absolute start point of the gantt, if configured.

    • data: The data from the tree root fetch for this header, if configured.

    Notes:

    • If you dont specify the "headers" property in a view, the default Insight Planning behaviour kicks in and the headers will be automatically calculated based on the views other properties.

    • If you define the "headers" property as an empty object, the view will have no headers at all.

    • You can define complete custom headers. If you do so, you need to define a scale that header should use.

    Configuration of tree-nodes for gantt

    Insight Planning requires some configurations to determine how to display nodes from a tree-configuration and the attributes to use for time-calculations.

    • temporal: Attributes to define a range of an Assignment. For Example the time-range a person is part of a team. Or the time range a team is planned to work on a job task. If no temporal configuration is present, the range will not be limited and therefore not be shown.

      • start: Attribute of type datetime to use as start of the range. The time will be interpreted as local time in the time-zone of the logged-in user. If the value of the record is null, the range is expected to have an open start.

      • finish: Attribute of type datetime to use as finish of the range. The time will be interpreted as local time in the time-zone of the logged-in user. If the value of the record is null, the range is expected to have an open end.

    • color: Color of the task or row as hexadecimal RGB value. You can use value or specify an attribute with the replacement syntax. Examples: "#AAAAFF" (light blue), "${activityStateColor}" (Color taken from the attribute activityStateColor)

    • planning: Planning specific properties

      • row: true Show node as row. Not allowed if a parent node is a task. If there is a temporal configuration and the row has a parent, the areas outside the assignment time-range between row and parent-row will be grayed out.

      • task: true Show node as task. Only allowed if the parent-node or the node itself is defined as a row. The start and finish of the task is calculated by the overlapping ranges of the nodes to the root. The label is the label of the node including the time range.

      • css: CSS class to apply to the gantt-task or gantt-row.

    Example

        "root": {
      "name": "team",
      "title": "Team",
      "label": "Team ${name}",
      "planning": {
        "row": true
      },
      "attributes": [...
      ],
      "children": [
        {
          "name": "teamAssignment",
          "temporal": {
            "start": "beginDate",
            "finish": "endDate"
          },
          "color": "${activityStateColor}",
          "planning": {
            "task": true,
          },
          "attributes": [...
          ]
        }
      ]
    }

    Context menu:

    If you specify a context menu for nodes shown as gantt-tasks or rows additional replacementParameters will be available to call a script. The root ganttrow and tasks will be reloaded upon changes.

    • ganttReferenceDate The reference date as shown above the gantt

    • ganttBeginDate The date, aligned to the viewprecision, of the position on the gantt row, where the right click was issued.

    Example

        {
      "label": "New Task",
      "action": "request",
      "request": {
        "url": "insight/action/business-suite/updateAssignment",
        "method": "POST",
        "params": {
          "resourceId": "${id}",
          "referenceDate": "${ganttReferenceDate}",
          "beginDate": "${ganttBeginDate}"
        },
        "headers": {
          "Accept": "application/json",
          "Content-Type": "application/json"
        }
      }
    },

    Moving gantt-tasks and Drag & Drop:

    To be able to move the dates of a gantt-task or if you want to move a gantt-task between the rows a Drag&Drop configuration between the row and the task is required. Drag and drop from outside the gantt are also possible.

    If you specify the optional clientProperty "allowRowSwitching": "false", a drag&drop between the gantt-rows will be prevented. The task can then only be moved on the time-axis.

    If dropped on the gantt the url-parameters of the called script can be parametrized with the following additional replacement parameters:

    • gantt.beginDate: The start of the task, aligned to the precision of the view. Example: "2007-12-03T00:00:00+02:00" This property will be available if:

      • the task is moved on the time-axis,

      • a drag & drop to the time-axis from outside the ganttt or

      • a drag & drop from between different rows.

    • gantt.endDate: The end of the task, aligned to the precision of the view. Will only be available, if the task is moved on the time-axis. Example: "2008-12-03T00:00+02:00"

    • gantt.referenceDate The reference date of the gantt (if not changed, the current day). Example: "2007-12-03T00:00+02:00"

    • src: The record of the task. Use to access the record of the task

    • dst: The record of the row. Use to access the record of the row

    Example of a Drag&Drop Configuration calling a script:

        {
      "src": "assignmentSickness",
      "dst": "person",
      "clientProperties": {
        "allowRowSwitching": false
      },
      "action": {
        "method": "POST",
        "url": "insight/action/business-suite/updateAssignment",
        "params": {
          "resourceId": "${dst.id}",
          "assignmentId": "${src.id}",
          "beginDate": "${gantt.beginDate}",
          "endDate": "${gantt.endDate}"
        },
        "headers": {
          "Accept": "application/json",
          "Content-Type": "application/json"
        }
      }
    }

    Example of a groovy-script for updating the start and finish date and moving between rows. The start and finish of the task is saved as local date and time in the fields beginDate and endDate:

        import java.time.*
    import java.time.temporal.*
    import java.time.format.*
    def toTemporal(dateString){
        return OffsetDateTime.parse(dateString).toLocalDateTime();
    }
    def updateAssignment(p){
        sql.withTransaction {
          if (p.beginDate!=null && p.endDate!=null){
            def begin,end;
            begin = toTemporal(p.beginDate);
            end = toTemporal(p.endDate);
            sql.executeUpdate("UPDATE Assignment SET beginDate=$begin, endDate=$end WHERE id=$p.assignmentId");
          }
          if (p.resourceId!=null){
            sql.executeUpdate("UPDATE Assignment SET resourceId=$p.resourceId WHERE id=$p.assignmentId");
          }
        }
    }
    updateAssignment(params);

    Capacity-planning configuration

    Capacity Planning in gannt is calculated by evaluating a sub-tree of a task.

    Capacity Planning uses the temporal start and finish configuration to determine the duration of an assignment. If you specify a temporal range on a treenode between the element to check and the subnodes with capacities, the range of the capacity will by limited by all configured nodes on the way.

    There are two different modes.

    • Availability Mode A Task will be shown with a red border, if a requirement can not be fulfilled by available resources.

    • Collision mode A Task will be show red if a record is found inside the range of a task on a sub-tree. Uses capacity type collision.

    If an availability is not reached, the task is marked with a red border and a Tooltip with the time ranges and the amount of the requirements not fulfilled. It will use the following additional properties:

    • capacity: Contains the configurations for capacity-planning
      • type: type of node for capacity-planning. The following values are valid:
        • requirement: Defines requirements that have to be fulfilled by available resources. Can be classified and quantified.
        • resource: A resource that will fulfill a requirement. Can be classified and quantified.
        • activity: Defines this node as activity. The first activity on the way from the task to the leaves will be expected to be the activity of the current node. If a resource is assigned to a different activity (sub node with type activity), the assigned resources will not be available for fulfilling the requirement.
        • collision: If a non-zero length record with this type is found on a sub tree of a task, the task will be shown red. No classification or quantity required.
      • classification: Required if type is requirement or resource. Defines the attribute with the classification of the requirement or resource.
      • quantity: Optional, if type is requirement or resource. Defaults to 1. Defines the attribute, where the amount of the requirement or the specified sources

    Tab

    To configure the label of the planning-tab use the following configuration.

        "tab": {
      "label": "Planung Teams"
    },

    Table

    Configure the table to show below the gantt:

        "table": {
      "tree": "jobtask"
    }

    All configurations from Insight Tables apply.

    Weekends

    Weekend days will automatically be visualized in a different color scheme. That scheme can be altered in the custom css file by changing the class:

        .gantt-custom-closed-weekend-date {
      // Your definitions
    }

    Highlights

    Inisght Planning can automatically fetch data to determine what columns to display in a different color scheme. The data will be retrieved from a different tree-configuration. This feature can be used to implement special visualization of vacation days for example.

    To support one specific category of highlightable dates, use the following notation:

        "menus": [
      {
        "label": "Calendar",
        "action": "planning",
        "icon": "icon-new-tab",
        "actionValues": [
          {
            "gantt": {
              "highlight": {
                "name": "Vacations",
                "tree": "public-holidays",
                "value": "date",
                "label": "name",
                "pattern": "YYYY-MM-DD",
                "css": "my-css-class-applied"
              }
            }
          }
        ]
      }
    ]

    You can also choose to support several different categories of highlightable dates. For example if you want to support public holidays, as well as facility based free days, you can use the following notation:

        "menus": [
      {
        "label": "Calendar",
        "action": "planning",
        "icon": "icon-new-tab",
        "actionValues": [
          {
            "gantt": {
              "highlight": [
                {
                  "category": "Vacations",
                  "tree": "public-holidays",
                  "value": "date",
                  "label": "name",
                  "pattern": "YYYY-MM-DD",
                  "css": "my-css-class-applied"
                },
                {
                  "category": "Other Vacations",
                  "tree": "public-holidays2",
                  "value": "date",
                  "label": "label",
                  "pattern": "YYYY-MM-DD",
                  "selected": false,
                  "css": "my-css-class-applied"
                }
              ]
            }
          }
        ]
      }
    ]

    If you want to highlight Date Ranges instead of specific dates, use the following notation:

        "highlight": [
      {
        "category": "Other Vacations",
        "tree": "public-holidays2",
        "from": "from",
        "to": "to",
        "label": "label",
        "pattern": "YYYY-MM-DD",
        "selected": false,
        "css": "my-css-class-applied"
      }
    ]

    You can mix date ranges with specific dates in one highlight category. If an entry has no "to" value, the "from" value will be interpreted as the concrete date.

    You can also define which of the highlight categories are active on Gantt Startup, setting "selected": "false" on that category. Default is true.

    To change the default color scheme of each highlight-category, simply add your own css classes to the customize.css file and reference them as shown above.

    Automatic Refresh

    Insight Planning supports automatic background refreshing. This will refresh all rows in the Gantt diagram. To reduce stress on the server, this feature is not enabled by default. You can enable this feature on a per tree basis with:

        "root": {**
      "planning": {
        "refresh": 30000
      }
    }

    You can also configure this feature in a planning action with:

        {
      "action": "planning",
      "actionValues": [
        {
          "gantt": {
            "refresh": 30000
          }
        }
      ]
    }

    You can also configure this feature in the Global Settings.

        Config = {
      components: {
        planning: {
          refresh: 30000
        }
      }
    }

    Filtering

    You can configure row filters for the Insight Planning Gantt Diagram.Those filters will display as a dropdown that the user can select from. All rows that match the selected filter values are displayed, all other rows are hidden. Multiple filters at the same time are supported. To enable row filters, configure filters at the "planning" menu action that opens the gantt diagram.

        {
      "label": "Employees",
      "action": "planning",
      "icon": "icon-new-tab",
      "actionValues": [
        {
          "single": true,
          "gantt": {
            "tree": "resource-person-activity",
            "filters": [
              {
                "name": "name",
                "label": "Name"
              },
              {
                "name": "description",
                "label": "Description"
              }
            ]
          }
        }
      ]
    }

    Please note that the configured filter's "name" is the attribute name of the record underlying the row model and has to match.

    Searches

    You can configure searches so that the user can search for specific task objects in the gantt diagram. The configured searches are close in functionality to the row filters mentioned above and fire against the configured underlying record's attribute.

        {
      "label": "Employees",
      "action": "planning",
      "icon": "icon-new-tab",
      "actionValues": [
        {
          "single": true,
          "gantt": {
            "tree": "resource-person-activity",
            "searches": [
              {
                "name": "name",
                "label": "Name"
              },
              {
                "name": "description",
                "label": "Description"
              }
            ]
          }
        }
      ]
    }

    Master Details

    You can configure a Master-Detail view for Insight Planning. When activated and a user clicks on any given task object in the gantt diagram, the details view of that task object will be loaded next to the gantt. Additionally, you can also configure the sizes of the gantt diagramm and appearing details view.

        {
      "label": "Employees",
      "action": "planning",
      "icon": "icon-new-tab",
      "actionValues": [
        {
          "single": true,
          "gantt": {
            "tree": "resource-person-activity",
            "filters": [
              {
                "name": "name",
                "label": "Name"
              },
              {
                "name": "description",
                "label": "Beschreibung"
              }
            ],
            "width": "twelve",
            "details": {
              "width": "four"
            }
          },
          "tab": {
            "label": "Planning Employees"
          },
          "table": {
            "tree": "jobtask",
            "width": "sixteen"
          }
        }
      ],
      "when": "onInit"
    }

    Custom Initial settings

    You can configure initial settings the gantt will use after loading.

    Date: The date the gantt will use as it's reference date after loading. If you dont specify this setting, the gantt will always load around the current Date.

        {
      "label": "Employees",
      "action": "planning",
      "icon": "icon-new-tab",
      "singleId": "planningEmployees",
      "actionValues": [
        {
          "gantt": {
            "initial": {
              "date": "2024-11-14T23:00:00.000Z"
            },
            "tree": "resource-person-activity"
          },
          "tab": {
            "label": "Planning Employees"
          },
          "table": {
            "tree": "jobtask"
          }
        }
      ]
    }

    You can also use the replace notation to set the initial date based on the clicked tree item's properties.

        "initial": {
      "date": "${planningStart}"
    }

    Search: The configured gantt search to fire automatically afterloading. If you dont have a matching search configured, Insight Planningwill throw an error.

        {
      "label": "Employees",
      "action": "planning",
      "icon": "icon-new-tab",
      "singleId": "planningEmployees",
      "actionValues": [
        {
          "gantt": {
            "initial": {
              "search": {
                "name": "activityState",
                "value": "Planung"
              }
            },
            "tree": "resource-person-activity"
          },
          "tab": {
            "label": "Planning Employees"
          },
          "table": {
            "tree": "jobtask"
          }
        }
      ]
    }

    You can also use the replace notation to set the initial search value based on the clicked tree item's properties.

        "initial": {
      "search": {
        "name": "activityState",
        "value": "${id}"
      }
    }

    Filter: Sets the configured filter automatically limiting the number of visual rows after loading. This does not prevent the actual loading of the filtered rows.

        {
      "label": "Employees",
      "action": "planning",
      "icon": "icon-new-tab",
      "singleId": "planningEmployees",
      "actionValues": [
        {
          "gantt": {
            "initial": {
              "filter": {
                "name": "activityState",
                "value": "C0345"
              }
            },
            "tree": "resource-person-activity"
          },
          "tab": {
            "label": "Planning Employees"
          },
          "table": {
            "tree": "jobtask"
          }
        }
      ]
    }

    You can also use the replace notation to set the initial filter value based on the first row's properties.

        "initial": {
      "search": {

        "value": "${id}"
      }

    Load Behaviour

    Loading lots of rows in Insight Planning can take a huge toll on client side performance, especially on older browser like IE.Having too many rows configured will take longer during the actual loading process and will result in bad rendering performance. If an Insight Planning instance is not limited to specific rows and instead will load all root rows of an Insight treeConfig, Insight Planning will automatically only load the first X rows. You can then navigate the currently loaded "frame" through the total amount of rootRows.The default amount for non-fullscreen loadable rows is 15, for fullscreen 40. You can override these settings with the following config:

        {
      "label": "Employees",
      "action": "planning",
      "icon": "icon-new-tab",
      "singleId": "planningEmployees",
      "actionValues": [
        {
          "gantt": {
            "rows": {
              "default": 15,
              "fullscreen": 40
            },
            "tree": "resource-person-activity"
          },
          "tab": {
            "label": "Planning Employees"
          },
          "table": {
            "tree": "jobtask"
          }
        }
      ]
    }

    If you load an Insight Planning instance with context objects, Insight Planning will automatically load the frame of root rows around the clicked context object. If you only want to load the actual clicked row and render it as the single row in the Gantt view, you can use the following toggle:

        {
      "label": "Employees",
      "action": "planning",
      "icon": "icon-new-tab",
      "singleId": "planningEmployees",
      "actionValues": [
        {
          "gantt": {
              "focusOnSelection": true
            },
            "tree": "resource-person-activity"
          },
          "tab": {
            "label": "Planning Employees"
          },
          "table": {
            "tree": "jobtask"
          }
        }
      ]
    }

    Jump Buttons

    Insight Planning supports specific buttons to jump to a preconfigured date. Example:

        "gantt": {
      "jumpTo": [
        {
          "label": "Start",
          "date": "${$payload.start}",
          "icon": "step backward"
        },
        { "label": "Heute", "icon": "clock" },
          {
            "label": "Ende",
            "date": "${$payload.end}",
            "icon": "step forward"
          }
      ],
    }

    Note: If you omit the "date" configuration, the button will result in a jump to the current date.

    Plugins

    Insight Planning has several technical plug-ins available for the Gantt diagram. You can enable and disable those plug-ins to trade performance for features.

    Example:

        {
      "label": "Employees",
      "action": "planning",
      "icon": "icon-new-tab",
      "singleId": "planningEmployees",
      "actionValues": [
        {
          "gantt": {
            "plugins": {
              "overlap": true
            },
            "tree": "resource-person-activity"
          },
          "tab": {
            "label": "Planning Employees"
          },
          "table": {
            "tree": "jobtask"
          }
        }
      ]
    }

    List of available plug-ins:

    • "overlap": Used for collision detection between several task objects. Default: false. Warning: Enabling this plugin has a huge toll on client performance with lots of rendered task objects.

    • "movable": Used for drag & drop actions on tasks inside the Gantt diagram. If the movable plugin is disabled, no tasks can be moved around. Default: true.

    Miscellaneous

    You can prevent users being able to resize tasks for each gantt, default being true.

        "gantt": {
      "allowResizing": false
    }

    You can hook into and change the actual temporal values for each record before they are being used for fetching data.

        "temporal": {
      "start": "zieltermin",
      "finish": "soll",
      "js": "if (moment(from).isBefore(moment(to))) { return { from: from, to: to }; } else { return { from: to, to: from }; }"
    }

    You can hook into the node.color setting and set the color with a js function.

        "color": {
      "js": "if(current.properties.zieltermin === null) { return '#ffbf00'} else if(moment(current.properties.soll).isSame(moment(current.properties.zieltermin), 'week')) { return '#ffbf00'} else if(moment(current.properties.soll).isBefore(moment(current.properties.zieltermin))) { return '#FF0000'; } else { return '#77df77'; }"
    }

    You can change the contextItems being used by Insight Planning, when starting a planning action from a context menu of an item. If you don't specify anything additional, Insight Planning will use all roots of the gantt Tree, by adding filterOnSelection, only the object the planning action was started on, will be used as a context item.

        "gantt": {
      "tree": "activity-standal-plan",
      "filterOnSelection": true
    }

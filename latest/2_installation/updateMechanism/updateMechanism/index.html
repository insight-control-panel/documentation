    Update-Mechanism (Maximo, OpenJet)¶

    Incremental update of business-objects to ElasticSearch for Searching works as follows:

    1. BusinessObject (i.e. Asset) is changed and stored in Maximo.
    2. LaunchPoint ESUPDATE-Script is trigged and stores a new record in table GISTX
    3. insight-Middleware requests every second changed data from EAM (i.e. Maximo)
    4. Connector reads and deletes all records in GISTX and response to request
    5. ChangeEvent is stored in in local database tempdb.RecentChangeEntity
    6. indexer requests ChangeEvents from local database and updates data in ElasticSearch

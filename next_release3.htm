V16.0
❑Section labelWidth: All labels get the same fix width in pixels.
❑Datetime-widget: Default changed! The "now"-action must configured if needed. clientProperties.nowAction = true. 
❑goto with parameters: Parameters can be passed to the details view. Special parameters are "headerLabel" and "headerLink". These two parameters can be used to control the header link of insight-mobile.
❑select-Action for selecting multiple root nodes
❑debug-Action
❑Checklist-widget for String or Text attributes. The checklist data is stored in the attribute as a json structure. 
❑SelectFromTree deep tree selection.
❑searches complex attributes: Attributes can be configured as complex objects to override existing attributes properties.
❑Master-Detail views for Insight Planning and Insight Tables.
❑Added initial settings for Insight Planning after loadup.
❑Attribute dirtyCheck: Include or exclude from dirty check of detail pages.
❑Documents imageEdit: Photos can be modified before upload. 
❑Documents inlineDelete: Default changed! Photos can't be deleted inline on the preview. The inline delete icon can be displayed using "inlineDelete" configuration. 
❑Attribute mustWrite: true: Attribute is written on save, even if the value is not changed. false: Attribute is never written on save.
❑Insight Planning headers: You can now configure all headers above the gantt diagram in Insight Planning.
❑Config.disableDefaultTab to suppress creating a StartCenter-Tab when working with Maximo
❑Config.disableMaximoUi to disable the Maximo-Ui integration completly, see chapter Client Configuration
❑remote.action() and remote.validate() for attribute-validation and attribute-control
❑Trees can be set to non-editable via rolesReadonly.

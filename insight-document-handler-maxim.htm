Document-Handler-Maximo
❑Stores the uploaded document on disk, creates a DOCLINK and set the relation to the owner.(WORKORDER, SR, ...)
❑In Maximo you have to create a DOCLINK-ObjectStructure with only the DOCLINK entity

❑In the insight.properties of wildfly/tomee you have to set the following properties
odocument.handler=maximo
odocument.doclink.url=http://localhost/maxrest/rest/os/DOCLINK
oeam.username=maxadmin
oeam.password=maxadmin
❑The file data can be processed in two different ways: (one of the properties must be set)
oStore the file to the file system. The insight middleware must be able to access the doclinks directory.
▪document.upload.folder=D:/documents
oSend the file data to the DOCLINKS object structure. No access to the doclinks directory needed. The file data must be loaded in the insight middleware memory.
▪document.doclink.mode=rest
❑There are also the properties configured in the backendParams part of the Document-Configuration
❑The Document-Configuration must have specific backendParams set:

 "backendParams": {

         "OWNERTABLE": "WORKORDER",

         "OWNERID": "${WORKORDERID}",

         "SITEID": "${userData.siteid}"

 }

oOWNERTABLE can be any Maximo-Entity, which can be used to attach DOCLINKS
oThere are additional backendParameters which can be added to change the default behavior.
oURLTYPE -> Default: "FILE"
oDESCRIPTION -> Default: "<file name> (mobile)"
oDOCTYPE -> Default: "Attachments"
oDOCUMENT -> Default: "MOBILE FILE"
oAdditional "DOCLINKS"-properties can be configured. These parameters are also saved.
